# 加法原语：一切运算的根本

## 三部曲的最终篇

前两篇文章分别讨论了数据的两个侧面：

1. **《整数对齐与分数概念》**——数据的**结构**问题：不同维度的度量怎么对齐，小数出现意味着什么
2. **《圆周归一化与空间对齐》**——数据的**空间**问题：如何把所有维度投影到圆周上，实现统一坐标系

但还有一个更根本的问题没说：**数据对齐了，放在那里，它自己不会动。**

1 就是 1，放在那里一万年还是 1。数字本身是静态的——它只记录状态，不产生变化。

要让数据"活"起来，需要一个最基本的操作：**加法**。

> **数静，加动。无加，数死不活。**

## 无论多复杂的算法，都是加法

这句话听起来像夸张，但逐层拆解就会发现，所有计算最终都归于加法。

### 算术层

- **乘法** = 连加。3 × 5 = 3 + 3 + 3 + 3 + 3
- **指数** = 连乘 = 加法的叠加。2³ = 2 × 2 × 2 = (2+2) + (2+2)
- **多项式** = 乘积累加。$ax^2 + bx + c$ 底层的计算单元：乘 → 加

### 线性代数层

- **矩阵乘法** $C = AB$：$c_{ij} = \sum_{k} a_{ik} \cdot b_{kj}$——对应元素相乘再**求和**
- **向量点积**：$\mathbf{a} \cdot \mathbf{b} = \sum_i a_i b_i$——乘积累**加**
- **所有线性变换**，最终都是加权**求和**

### 深度学习层

- **一个神经元**：$y = \sigma(\sum_i w_i x_i + b)$——加权**加**法 + 激活
- **全连接层** = 矩阵乘法 = 大量加法的集合
- **卷积层** = 滑动窗口内的加权**求和**
- **Transformer Attention**：$\text{Attention}(Q,K,V) = \text{softmax}(QK^T)V$——不管中间的 softmax 多复杂，最后一步还是**加权求和**
- **残差连接**：$\mathbf{x} + \text{FFN}(\mathbf{x})$——最简单的**加**
- **梯度下降**：$\theta_{t+1} = \theta_t - \eta \nabla L$——参数更新就是**加法**（加上负梯度）
- **Batch Normalization**：减去均值、除以标准差——减和除，但最终恢复变换时还是**加**（$\gamma \cdot \hat{x} + \beta$）

从最简单的感知机到最大的 LLM，前向传播、反向传播、参数更新——**每一层都在做加法**。

### 分析层

- **积分** = 无穷小量的无限**累加**
- **傅里叶级数** = 正弦波的加权**叠加**
- **泰勒展开** = 多项式的逐项**累加**
- **全概率公式** = 条件概率的加权**求和**

### 东方数术层

- **奇门遁甲**：排盘就是干支的叠加运算。值符随旬首飞布、值使随地支飞转——每一步都是对一个静态符号加上一个偏移量。**九宫飞布的本质，是对九宫格坐标的加法投影。**
- **八字**：大运是月柱的步进叠加（阳男阴女顺推，阴男阳女逆推，每步加 10 天干或 12 地支）。流年是年柱的逐年加 1。**命理推步的本质，是干支的周期加法。**
- **《周易》**：太极生两仪，两仪生四象，四象生八卦…… **每一层叠加都是"二进制的加法展开"**：两仪（0/1）→ 四象（2 bit）→ 八卦（3 bit）→ 六十四卦（6 bit）。卦序的排列，就是布尔加法在圆圈上的遍历。
- **纳甲配爻**：乾震坎艮坤巽离兑各配 6 爻，每爻配上不同天干——本质是把天干"**加**"到爻位上，让静态的卦有了干支属性的动态关系。
- **五行生克**：生 = **增益**（加），克 = **减损**（减）。五行本身是静态的标签（木火土金水），但生克关系一旦建立，就变成了一个运算系统——**五行不是一个分类系统，它是一个加法群**。
- **六十甲子**：天干 × 地支 = LCM(10, 12) = 60。**六十甲子本身就是乘加运算的产物**——把两个度量周期"乘"在一起，获得一个更大的周期，然后在那个大周期里逐次**加** 1 遍历。

## 数据与运算的二元性

从这里可以抽象出一个更本质的认识：

> **任何一个计算系统，都可以拆为「数据」和「运算」两个维度。**

| | 数据（静） | 运算（动） |
|--|----------|----------|
| 哲学 | 存在 | 变化 |
| 状态 | 是什么 | 变成什么 |
| 东方 | 阴（静） | 阳（动） |
| 例子 | 干支符号、卦象、数字 | 加法、飞布、生克 |
| 现代 | Tensor、参数、权重 | 前向传播、梯度更新 |

**数据是静态的，运算让数据产生关系。**

一个数字在那里，一万年不动——它就是它自己。但一旦你给它加上另一个数，它就变了，就和外部世界建立了联系。**加法是运算的最小单位，是数据从孤立走向关系的第一步。**

## 加法的玄学时刻：数据世界，不动就是死

到这里有一个跳脱的观察：

> **数据世界不怕变，怕不动。不动就是死数据。**

- 一个神经元如果权重不更新——它就是个固定映射，学不了任何东西
- 一个 LLM 如果参数不做加法（前向传播本质上是大量加权加法）、不做加法更新（梯度下降）——它就是一堆浮点数，没有任何智能
- 奇门遁甲的排盘如果不去"飞"——值符值使不动——它就是一张静态的图，看不出吉凶祸福的趋势
- 八字的推运如果不做"加"——大运不推、流年不加——就只是一张出生时的快照，看不出人生起伏
- 五行的生克如果不动——木永远是木、火永远是火——就没有"木生火、火生土"的能量流转

**一切"活"的系统，底层都有加法在持续运行。**

加法让一切运转起来——无论是神经网络里的百亿次浮点运算，还是八字排盘里的一次干支步进。**运算 = 系统在呼吸。**

## 结语

前两篇文章回答了"数据怎么对齐、怎么归一"，这篇回答了"对齐之后呢？"——**加。**

无论多复杂的算法——深度学习、Transformer、LLM、奇门遁甲、八字、纳甲、五行生克——在最底层跑的都是加法。加法是运算的原语（primitive），是数据从静态到动态的第一次跃迁。

> **数静，加动。无加，数死不活。**
>
> **形而上者谓之道，形而下者谓之器。加法既非道亦非器——它是道器之间的那一步。**
